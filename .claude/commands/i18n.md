# Internationalization

Make sure there are no hardcoded strings in the code. Use the i18n system to handle translations.

## Guidelines

- Use `vue-i18n` for all user-facing text
- Import the translation function: `import { t } from '@services/i18n'`
- Translation files are in `./locales/*.json`
- English is the main language - only add English translations when creating new features

## Checking i18n

Run the i18n check tool to verify all translations:
```bash
./tools/i18n_check.ts
```

This tool will:
- Detect missing translation keys
- Find unused translation keys
- Report keys with changed English text

## Fixing i18n Issues

To automatically fix issues (add missing translations, remove unused keys):
```bash
./tools/i18n_check.ts --fix
```

Note: The `--fix` flag will use an LLM to translate missing keys to other languages. Only use this when explicitly requested by the user.

## Important

- Never run `./tools/i18n_check.ts --fix` without explicit user request
- Only add English translations for new features
- Do not add translations for other languages unless specifically asked
